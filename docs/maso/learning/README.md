# MASO Learning Module

**A structured guide to understanding and applying the Multi-Agent Security Operations Framework.**

This module distills MASO into its essential principles, concepts, and practical application patterns. It is designed for architects, engineers, security professionals, and anyone building or governing multi-agent AI systems.

## Module Structure

| Module | Focus | Audience |
|--------|-------|----------|
| [1. Foundations](01-foundations.md) | What MASO is, why it exists, and the core mental model | Everyone |
| [2. The Three-Layer Defence](02-three-layer-defence.md) | Guardrails, LLM-as-Judge, and Human Oversight — how they work together | Architects, Engineers |
| [3. PACE Resilience](03-pace-resilience.md) | Designing for failure with Primary, Alternate, Contingency, Emergency | Architects, Operations |
| [4. The Seven Control Domains](04-seven-control-domains.md) | What each domain protects and why it matters | Security, Governance |
| [5. Emergent Risks](05-emergent-risks.md) | Risks unique to multi-agent systems — epistemic, coordination, and compound failures | Everyone |
| [6. Applying MASO to Your Solution](06-applying-maso.md) | Practical guidance for designing and building with MASO | Engineers, Architects |
| [7. Key Takeaways and Quick Reference](07-quick-reference.md) | Condensed principles, checklists, and decision aids | Everyone |

## How to Use This Module

**If you have 15 minutes:** Read [Module 1: Foundations](01-foundations.md) and [Module 7: Quick Reference](07-quick-reference.md).

**If you are designing a system:** Work through Modules 1 through 6 in order, then keep Module 7 as a desk reference.

**If you are reviewing or governing a system:** Focus on Modules 1, 4, 5, and 7.

## Prerequisites

No prior knowledge of MASO is required. Familiarity with LLM-based applications and basic security concepts is helpful.

## Relationship to Full Framework Documentation

This learning module is a teaching companion, not a replacement for the full [MASO Framework](../README.md). It covers principles and concepts to build understanding. For control specifications, implementation details, and compliance mappings, refer to the primary framework documentation.
